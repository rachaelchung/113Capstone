# Reflection

1. **Describe your process.** At a high level, how did you go about creating this project?

For this project, I used Claude to start me and Cursor to edit thoroughly. I actually hadn't used Cursor before as much as I did for this project. I deferred to Claude for UI while Cursor worked on everything else. 

I wrote a SPEC for this project, but ended up having to forgo a lot of the things I wanted. However, it was important to me I work in sections (game, assignments, chrome extension). Since I had a spec that outlined what the end product would hopefully be, working on indivdual points felt less overwhelming. The project always knew what the main goal was going to be and worked toward that end.

I did mostly human reviewing, relying on myself to review my work. I did a little review with my assignments tracker, but really could only rely on myself to give feedback.

Once I was sure everything worked locally, I deployed my app to GitHub Pages and Render.

2. **What AI tools and strategies did you use?** What models did you choose, and were you using them in a browser chat, in something like Cursor, in the command line, etc? Did you use an agentic process like in HW8, or something else?

I used Sonnet 4.5 in Claude and Composer 2 and Opus 4.7 (only a little) in Cursor. I pretty much did eveything within Cursor this time around because I knew I was going to have a lot of files. I had a mixed agentic process, where I reviewed it myself, but still wrote a SPEC so the model new what was going on at all times. Since I worked on the project piece by piece and had the AI implement and merge things together, it may have felt a little agentic, but not in the same process we used in HW8.

3. **Why did you make the choices above?**

Claude is my best friend with UI and UX. I knew if I started in Claude and it gave me a baseline for the UI, I could focus on just using Cursor with the foundations of what Claude gave me. (This sort of fell apart, and Cursor was not great at all at copying design from Claude. But that is why I did it the way I did initially.)

4. **What changed from your pre-113 approach?** Compare your approach to how you might have approached the project before 15-113. What has changed, if anything? Do you feel more or less well-equipped to create projects like this within a reasonable time frame?

Before 15-113, I don't think I had any experience with coding with AI ("vibe coding"). I'm not sure I could have even thought up a project like this without all the knowledge I gained at a high level about databases and backends and everything else. If someone asked me to recreate this before I took 113, I wouldn't even know how to prompt an AI. 

Now I do stand by writing a SPEC, and I like AI's that work inside my codespace. I think before I probably would've just been doing everything in one file, because that's all an AI would read at one time, and it would be incredibly messy and broken.

I feel really well-equipped to create projects like this in a good time frame. Today, when I have something I need to create, I even can begin thinking about how I could easily create an app for that or a website for that. Because of this class, I even created a website for an event we needed to get info out for in one day, letting the event run way more smooth.

5. What might you do with this project with more time?

I would do lots more with this project. For one, I would create actual art: for the creatures or for the backgrounds. I would make the creatures more fun and more interactible so that you actually want to talk to them and do lots more to make the "game" feel a bit more like an actual game. I would also expand beyond just a chrome extension and create an actual app in something like Electron to work across applications, not just chrome. I would probably also continue working on the CSS for the assignments categories since I am still not in love with it and it still has a couple bugs.
